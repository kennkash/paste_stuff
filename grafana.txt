Yes — here are simple, end-user-friendly descriptions you can use for the panels shown in your dashboard.

API Information row

API Information
A summary view of how the selected APIs are being used and how they are performing.

Panels

Total Requests
Shows the total number of API requests received in the last 24 hours.

Requests Count
Shows how many requests each API endpoint received in the last 24 hours.

Requests Average Duration
Shows the average time each API request takes to complete.

Percent of 2xx Requests
Shows the share of requests that completed successfully for each endpoint.

Percent of !2xx Requests
Shows the share of requests that did not return a successful response, broken out by endpoint and status code.

PR 99 Requests Duration
Shows the slowest request times for each endpoint, based on the top 1% of requests.

Request time
Shows how request time changes over time for each endpoint.

Request Per Sec
Shows how many requests each endpoint is receiving per second.

Average Request Duration per 10min
Shows the average response time for the selected apps, updated in 10-minute intervals.

API Response Time
Shows response times across endpoints in a heatmap so it is easy to spot slower APIs.

Error Response By Endpoint
Shows whether any non-success responses have occurred for an endpoint in the last 10 minutes.

200 http_requests
Shows the trend of successful HTTP requests over time.

Requests per 5min Interval
Shows how many requests were made every 5 minutes, grouped by app, endpoint, and status.

Logs Volume
Shows the number of logs over time by log level, and can be clicked to drill into the matching logs.
